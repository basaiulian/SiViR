1. Introduction

SiViR(Similar Video Retriever) is a Web Application that offers you the possibility of searching videos on multiple platforms such as: YouTube, Vimeo, Instagram.

2. Technologies used

2.1 HTML5

HTML 5 is a markup language used for structuring and presenting content on the World Wide Web. It was the fifth and last major version of HTML that is a World Wide Web Consortium (W3C) recommendation.

2.2 CSS

CSS stands for "Cascading Style Sheet." Cascading style sheets are used to format the layout of Web pages. They can be used to define text styles, table sizes, and other aspects of Web pages that previously could only be defined in a page's HTML.

2.3 PHP

PHP (recursive acronym for PHP: Hypertext Preprocessor) is a widely-used open source general-purpose scripting language that is especially suited for web development and can be embedded into HTML.

2.4 XML

XML stands for "Extensible Markup Language.". XML is used to define documents with a standard format that can be read by any XML-compatible application.

2.5 JavaScript

JavaScript(JS) is a programming language commonly used in web development. It is a lightweight, interpreted, or just-in-time compiled programming language with first-class functions.

2.6 AJAX

AJAX (Asynchronous JavaScript and XML) allows web pages to be updated asynchronously by exchanging data with a web server behind the scenes. This means that it is possible to update parts of a web page, without reloading the whole page.

2.7 MySQL

MySQL is a freely available open source Relational Database Management System (RDBMS) that uses Structured Query Language (SQL). SQL is the most popular language for adding, accessing and managing content in a database.

2.8 HTTP Methods

There are two ways the browser client can send information to the web server:

 The GET method sends the encoded user information appended to the page request.

 The POST method transfers information via HTTP headers.

2.9 REST

Representational state transfer (REST) is a software architectural style that defines a set of constraints to be used for creating Web services. Web services that conform to the REST architectural style, called RESTful Web services, provide interoperability between computer systems on the Internet.

2.10 API

Application programming interface (API) is a set of rules that allows one piece of software application to talk to another. Those "rules" can include the create, read, update and delete operations (CRUD). The API is created from models (database tables), endpoints and the connection to the database. Each model is represented by a class which contains attributes and methods. At a request to an endpoint in the API, certain attributes will be set and certain methods will be called. In our application calls are made to YouTube API, Instagram API and Vimeo API, and the answers obtained will be taken and processed in JSON format. The information extracted from the answers is sent to the interface so that it can be viewed, modified or deleted by the user.

3. Application architecture

The Model-View-Controller (MVC) is an architectural pattern that separates an application into three main logical components: the model, the view, and the controller. Each of these components are built to handle specific development aspects of an application.

3.1 Model

The central component of the pattern. It is the application's dynamic data structure, independent of the user interface. It directly manages the data, logic and rules of the application.

3.2 View

Any representation of information such as a chart, diagram or table. Multiple views of the same information are possible, such as a bar chart for management and a tabular view for accountants.

3.3 Controller

The controller is that part of the application that handles the user interaction. The controller interprets the mouse and keyboard inputs from the user, informing model and the view to change as appropriate.

API folder containts endpoints and models.

    instagram.php - An object of type InstagramModel is created and the function corresponding to the search is called.

    user.php - Endpoints for each function in the UserModel.

    video.php - Create an object of each type (Instagram / Vimeo / Youtube) and call the search function of each.

    vimeo.php - An object of type VimeoModel is created and the function corresponding to the search is called.

    youtube.php - An object of type YoutubeModel is created and the function corresponding to the search is called.

    MODELS folder

      instagram.php - contains the InstagramModel class where the call to the Instagram API is made.

      user.php - Contains user information and functions for Login, Register, Delete, checkingUser, checkAdmin, inserting user preferences and retrieving the necessary information.

      video.php - Retrieve information from the user and call the appropriate requests according to the user's preferences.The response from the API is saved in an array so that it can be processed.

      vimeo.php - Contains the VimeoModel class where the call to the VIMEO API is made.

      youtube.php - Contains the YoutubeModel class where the call to the Youtube API is made.

CONTROLLERS folder

    vendor - Vimeo and Instagram library

    control_panel.php - Contains the call to the API to get all registered users.

    favourites.php - Includes the corresponding file in the view to make the connection between the interface and the functionality.

    home.php - Includes the corresponding file in the view to make the connection between the interface and the functionality.

    login.php UserController handles requests to API and validates the user.

    logout.php Ends the user login session.

    register.php RegisterController handles requests to Api and creates the user if it is possible.

PUBLIC Folder

      The register_style.css has the role of formatting HTML from register.php document.

    js folder

      The home.js Displays search information and provides video functionality.

      The favourite.js Displays information about the user's latest accesses and provides video functionality.
